Clone World

Description

Clone-World is a system that allow you to interactively browse code clones in a software system.

Documentation

JSON Files format

  1. classes.json
    it contains a single list about information about classes
    1. for each object of list
      • type: the class type (1, 2, 3)
      • id: the class id
      • cloneCount: how many clones fragments in this class
  2. files.json
    it contains a single list about information about files
    1. for each object of list
      • path: the file path
      • cloneCount: how many clones fragments in this file
  3. clones.json
    it contains a single list about information about clones
    1. for each object of list
      • id: the clone id
      • chainId: the chain id of this clone fragment
      • classType: the type of this clone (1, 2, 3)
      • classId: the class id of this clone
      • filePath: the file path of this clone
      • startLine: the start line of this clone in the source code
      • endLine: the end line of this clone in the source code
      • sumChangeCount: the sum of change counts in all revisions
      • finalRevision: the last revision that this clone exsits
      • x: the x coordinate of the scatter plot
      • y: the y coordinate of the scatter plot
      • DC: the distinct color of the directory of this clone
      • CC: the distinct color of the class of this clone
  4. system.json
    it contains information of system:
    • name: the name of the system (carol)
    • finalRevison: the last revision in the dataset
  5. carol.json
    it contains information of all clone chains in system:
    • max_revision: end revision of system
    • min_revision: start revision of system
    • max_changecount: highest sum of changecount for all chains
    • max_instance_changecount: highest changecount for clone instance for all clone instance in all revision
    • system_name: name of system
    • t1_clone_evolution: array of all clone chains in type 1
      1. for each object in array:
        • startRevision: start revision of chain
        • endRevision: end revision of chain
        • sumChangeCount: sum of changecount of this chain
        • name: chain id
        • type: type of chain
        • supportCountMap
          this object contains support count of current chain compare to another chain
          • another chain id : support count between current chain & another chain
        • crossBpundaryCountMap
          this object contains cross boundary count of current chain compare to another chain
          • another chain id : cross boundary count between current chain & another chain
        • baseCase
          this object contains information about current clone chain in startRevision
          • filePath: the file path of this clone
          • startLine: the start line of this clone in the source code
          • endLine: the end line of this clone in the source code
          • changecount: the changecount of this clone
          • changetype: type of chain of this clone("a"-> addition, "d"-> deletion, "c"--> correction, "-1" --> not recorded)
          • cloneclass: the clone class id of this clone
        • cloneInstances
          this object contains information about current clone chain in revisions after start revision
          • for each object in cloneInstances object: it only contains fields which is different compare to baseCase object
            In example:
            • endLine: 14
            • startLine: 7
            • cloneclass: 4
    • t2_clone_evolution: array of all clone chains in type 2
      in same format of t1_clone_evolution
    • t3_clone_evolution: array of all clone chains in type 3
      in same format of t3_clone_evolution
  6. colors.json
    array of 1000 colors used to color clone chains in parallel coordiante and force layout
